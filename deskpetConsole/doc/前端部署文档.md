# 部署指南

本项目的构建产物通常部署在 Nginx 等 Web 服务器上。

## 1. 构建项目

在本地或 CI/CD 环境中执行构建命令：

```bash
# 安装依赖
npm install

# 执行构建
npm run build
```

构建完成后，会在项目根目录生成 `dist/` 文件夹。

## 2. Nginx 配置

将 `dist/` 目录下的所有文件上传至服务器（例如 `/usr/share/nginx/html`）。

推荐的 Nginx 配置如下：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /usr/share/nginx/html;
    index index.html;

    # 开启 gzip 压缩
    gzip on;
    gzip_min_length 1k;
    gzip_buffers 4 16k;
    gzip_comp_level 5;
    gzip_types text/plain application/javascript application/x-javascript text/css application/xml text/javascript application/x-httpd-php image/jpeg image/gif image/png;
    gzip_vary on;

    # 前端路由支持（防止刷新 404）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端接口代理
    location /api/ {
        proxy_pass http://backend-service:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # 静态资源缓存控制
    location /assets/ {
        expires 7d;
        access_log off;
    }
}
```

## 3. 常见问题

### 页面刷新 404
Vue Router 使用 history 模式，必须在 Nginx 中配置 `try_files $uri $uri/ /index.html;`，确保所有非静态资源请求都回退到 `index.html`。

### 接口 404
确认后端服务地址正确，并检查 Nginx 的 `proxy_pass` 末尾是否有 `/`（影响路径重写）。
