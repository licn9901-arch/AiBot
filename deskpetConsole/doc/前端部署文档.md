# 部署指南

本项目包含两个前端应用，构建产物均部署在 Nginx 等 Web 服务器上。

## 1. 构建项目

```bash
# 管理后台
cd deskpetConsole/manager
npm install
npm run build
# 产物目录：manager/dist/

# 用户端
cd deskpetConsole/user-frontend
npm install
npm run build
# 产物目录：user-frontend/dist/
```

## 2. Nginx 配置

将两个项目的 `dist/` 目录分别上传至服务器。推荐配置如下：

```nginx
# 管理后台（admin.your-domain.com）
server {
    listen 80;
    server_name admin.your-domain.com;

    root /usr/share/nginx/manager;
    index index.html;

    gzip on;
    gzip_min_length 1k;
    gzip_buffers 4 16k;
    gzip_comp_level 5;
    gzip_types text/plain application/javascript application/x-javascript text/css application/xml text/javascript application/json;
    gzip_vary on;

    # 前端路由支持（防止刷新 404）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # 后端接口代理
    location /api/ {
        proxy_pass http://backend-service:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # 静态资源缓存
    location /assets/ {
        expires 7d;
        access_log off;
    }
}

# 用户端（app.your-domain.com）
server {
    listen 80;
    server_name app.your-domain.com;

    root /usr/share/nginx/user-frontend;
    index index.html;

    gzip on;
    gzip_min_length 1k;
    gzip_buffers 4 16k;
    gzip_comp_level 5;
    gzip_types text/plain application/javascript application/x-javascript text/css application/xml text/javascript application/json;
    gzip_vary on;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://backend-service:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /assets/ {
        expires 7d;
        access_log off;
    }
}
```

### 单域名部署（可选）

如果使用同一域名，可通过路径前缀区分：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 用户端（默认）
    location / {
        root /usr/share/nginx/user-frontend;
        try_files $uri $uri/ /index.html;
    }

    # 管理后台
    location /admin/ {
        alias /usr/share/nginx/manager/;
        try_files $uri $uri/ /admin/index.html;
    }

    # 后端接口代理
    location /api/ {
        proxy_pass http://backend-service:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /assets/ {
        expires 7d;
        access_log off;
    }
}
```

## 3. 常见问题

### 页面刷新 404
Vue Router 使用 history 模式，必须在 Nginx 中配置 `try_files $uri $uri/ /index.html;`，确保所有非静态资源请求都回退到 `index.html`。

### 接口 404
确认后端服务地址正确，并检查 Nginx 的 `proxy_pass` 末尾是否有 `/`（影响路径重写）。

### 移动端访问
用户端已内置响应式适配，移动端浏览器访问时自动切换为 Vant UI + 底部 TabBar 布局，无需额外配置。
