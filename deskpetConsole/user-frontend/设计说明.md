# DeskPet 桌宠机器人管理平台 UI设计需求说明
## 一、基础信息说明
1.  **产品名称**：DeskPet 桌宠机器人管理平台
2.  **产品定位**：面向桌宠机器人终端用户的移动端设备管理平台，核心实现设备状态监控、授权激活、个人信息管理、设备快捷操作，是用户管控智能桌宠硬件的核心入口。
3.  **适配终端**：移动端竖屏设计，以375px宽度为基准设计，兼容主流手机尺寸的响应式适配。
4.  **目标用户**：DeskPet桌宠机器人的购买与使用用户。
5.  **核心页面清单**：首页、设备列表页、激活授权码页、个人中心页，共4个核心页面。

---

## 三、整体设计规范要求
### 1. 视觉风格定位
- 核心调性：简约科技感+亲和友好感，贴合「桌宠」智能陪伴产品的定位，避免冰冷的工业后台风格，兼顾轻量化、扁平化与微质感，用圆角、轻渐变、弱阴影打造层次，拒绝过度设计。
- 适配要求：以375px宽度为设计基准，做好响应式适配，保证主流手机尺寸下显示一致。

### 2. 色彩体系规范
- 品牌主色：以原型中的蓝色为核心主色，需定义主色、深浅变体、禁用态色值，用于核心按钮、选中态、高亮元素、品牌视觉。
- 功能色规范：
  - 成功/在线态：绿色系，用于在线状态、成功提示、勾选图标；
  - 错误/离线态：红色系，用于离线状态、错误提示、关闭图标；
  - 中性色：定义白、浅灰、中灰、深灰、黑的完整色阶，用于背景、卡片、文案、分割线，保证页面层级清晰。
- 渐变规范：统一定义Banner区的蓝色渐变起止色值，全产品品牌视觉统一。

### 3. 字体与排版规范
- 字体选择：中文优先使用苹方/思源黑体，英文使用SF Pro/DIN，无衬线字体保证跨平台显示一致性。
- 字号层级：定义完整的字号体系，严格区分大标题、次级标题、正文、辅助文案、标签文案，对应明确的字重（标题用Medium/Bold，正文用Regular，辅助文案用Light）。
- 间距规范：以8px为最小基准单位，统一定义模块间距、卡片内边距、行高（正文1.4-1.5倍，标题1.2倍），保证全页面排版整齐统一。

### 4. 组件与交互规范
- 通用组件统一：统一定义按钮（主/次/文字/图标按钮，含默认、hover、点击、禁用态）、输入框（含默认、聚焦、禁用态）、标签、卡片、Tab栏的样式规范，全产品复用，保证视觉一致性。
- 交互反馈：所有可点击元素必须有点击态反馈；刷新、提交、状态切换需搭配200-300ms的流畅动效；输入、加载、报错均有明确的视觉反馈。
- 全状态设计：需补充设计设备列表空状态、页面加载loading态、网络/操作错误态、组件禁用态的对应样式。

### 5. 细节与拓展要求
- 图标规范：所有功能图标、Tab图标统一设计风格（线性/面性统一、圆角/粗细统一）。
- 拓展性：设计需预留后续功能拓展空间（如设备详情控制、多分组管理、多设备批量操作等），组件与布局具备可复用、可拓展性。
- 输出要求：设计稿需标注完整的尺寸、色值、字号、间距，按规范输出切图，建议同步提供可点击的交互原型演示。
