# DeskPet 线上部署指南

## 一、服务器要求

| 项目 | 最低配置 | 推荐配置 |
|------|---------|---------|
| CPU | 2 核 | 4 核 |
| 内存 | 4 GB | 8 GB |
| 磁盘 | 40 GB SSD | 100 GB SSD |
| 系统 | Ubuntu 20.04+ / CentOS 7+ | Ubuntu 22.04 |
| 网络 | 公网 IP，开放 80/443/1883 端口 | 同左 |

## 二、部署架构

```
                     互联网
                       │
               ┌───────┴───────┐
               │ Nginx (原生)   │  ← 统一入口，SSL 终止，静态文件托管
               │ :80 / :443    │
               └──┬──┬──┬──┬───┘
                  │  │  │  │
   ┌──────────────┘  │  │  └──────────────┐
   │                 │  │                 │
   ▼                 ▼  ▼                 ▼
 静态文件          静态文件              反向代理
 用户前端          管理后台
 /var/www/...      /var/www/...
                     │  │
              ┌──────┘  └──────┐
              ▼                ▼
   ┌─── Docker Compose ────────────────┐
   │                                   │
   │  pet-core    pet-ai    mqtt-gw    │
   │  :8080       :8090     :1883      │
   │  (127.0.0.1) (127.0.0.1)  (公网) │
   │       │                           │
   │  ┌────┼────────────┐              │
   │  ▼    ▼            ▼              │
   │  DB   TimescaleDB  Redis          │
   │  (内部) (内部)      (内部)         │
   └───────────────────────────────────┘
```

- Nginx 原生安装在宿主机，直接 serve 前端静态文件，反向代理后端 API
- 后端服务（pet-core / pet-ai）端口绑定 `127.0.0.1`，不对外暴露
- 数据库、Redis 不暴露端口，只在 Docker 内部网络通信
- MQTT 端口 1883 直接对外（设备接入）

## 三、服务器初始化

### 3.1 安装 Docker

```bash
curl -fsSL https://get.docker.com | sh
sudo usermod -aG docker $USER
# 重新登录使 docker 组生效
```

验证：

```bash
docker compose version
```

### 3.2 编译安装 Nginx

```bash
# 安装编译依赖
sudo apt update
sudo apt install -y build-essential libpcre3 libpcre3-dev zlib1g-dev libssl-dev libgeoip-dev

# 下载 Nginx 源码
cd /usr/local/src
sudo wget https://nginx.org/download/nginx-1.26.2.tar.gz
sudo tar -xzf nginx-1.26.2.tar.gz
cd nginx-1.26.2

# 编译安装
sudo ./configure \
    --prefix=/etc/nginx \
    --sbin-path=/usr/sbin/nginx \
    --modules-path=/usr/lib64/nginx/modules \
    --conf-path=/etc/nginx/nginx.conf \
    --error-log-path=/var/log/nginx/error.log \
    --http-log-path=/var/log/nginx/access.log \
    --pid-path=/var/run/nginx.pid \
    --lock-path=/var/run/nginx.lock \
    --with-http_ssl_module \
    --with-http_v2_module \
    --with-http_realip_module \
    --with-http_gzip_static_module \
    --with-http_stub_status_module \
    --with-stream

sudo make && sudo make install
```

创建 systemd 服务：

```bash
sudo tee /etc/systemd/system/nginx.service > /dev/null <<'EOF'
[Unit]
Description=Nginx HTTP Server
After=network.target

[Service]
Type=forking
PIDFile=/var/run/nginx.pid
ExecStartPre=/usr/sbin/nginx -t
ExecStart=/usr/sbin/nginx
ExecReload=/bin/kill -s HUP $MAINPID
ExecStop=/bin/kill -s QUIT $MAINPID
PrivateTmp=true

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable nginx
```

验证：

```bash
nginx -v
sudo systemctl start nginx
curl http://localhost
```

### 3.3 安装 Node.js（用于构建前端）

```bash
# 使用 NodeSource 安装 Node.js 20
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt install -y nodejs

node -v
npm -v
```

### 3.4 安装 JDK 17 和 Maven（如果在服务器上构建）

本地构建镜像再推送的方式不需要在服务器上安装这些。

```bash
sudo apt install -y openjdk-17-jdk maven
java -version
mvn -version
```

## 四、部署步骤

### 4.1 准备部署目录

```bash
sudo mkdir -p /opt/deskpet
sudo mkdir -p /var/www/deskpet/user-frontend
sudo mkdir -p /var/www/deskpet/manager
sudo chown -R $USER:$USER /opt/deskpet /var/www/deskpet
```

### 4.2 上传部署文件

在本地执行（将部署配置传到服务器）：

```bash
scp deploy/docker-compose.prod.yml user@your-server:/opt/deskpet/docker-compose.yml
scp deploy/.env.prod.example user@your-server:/opt/deskpet/.env
scp deploy/gateway-config.yaml user@your-server:/opt/deskpet/
scp deploy/timescaledb-init.sql user@your-server:/opt/deskpet/
```

### 4.3 配置环境变量

在服务器上编辑 `/opt/deskpet/.env`：

```bash
cd /opt/deskpet
vim .env
```

必须修改的项：

```bash
# 数据库密码 - 使用强密码
DB_PASSWORD=your_strong_password_here
TSDB_PASSWORD=your_strong_password_here

# Redis 密码
REDIS_PASSWORD=your_redis_password_here

# 内部通信 Token - 随机生成
INTERNAL_TOKEN=$(openssl rand -hex 32)

# AI 模块 API Key
OPENAI_API_KEY=sk-your-real-key

# 如果使用镜像仓库
REGISTRY=registry.cn-hangzhou.aliyuncs.com/yournamespace/
TAG=v0.1.0
```

### 4.4 构建与部署后端（Docker）

有两种方式：

#### 方式 A：本地构建 + 推送镜像（推荐）

在本地机器上：

```bash
# 登录镜像仓库
docker login registry.cn-hangzhou.aliyuncs.com

# 构建并推送
cd deploy
# Windows
build.bat v0.1.0 registry.cn-hangzhou.aliyuncs.com/yournamespace/
# Linux/Mac
./build.sh v0.1.0 registry.cn-hangzhou.aliyuncs.com/yournamespace/
```

在服务器上：

```bash
cd /opt/deskpet
docker login registry.cn-hangzhou.aliyuncs.com
docker compose pull
docker compose up -d
```

#### 方式 B：服务器上直接构建

```bash
# clone 代码
cd /opt
git clone https://your-repo-url.git deskpet-src
cd deskpet-src

# 构建镜像
cd deploy
chmod +x build.sh
./build.sh latest

# 启动
cd /opt/deskpet
docker compose up -d
```

### 4.5 构建与部署前端

前端不走 Docker，直接构建静态文件放到 Nginx 目录。

#### 在服务器上构建

```bash
# 如果代码已在服务器上
cd /opt/deskpet-src

# 构建用户前端
cd deskpetConsole/user-frontend
npm ci
npm run build
cp -r dist/* /var/www/deskpet/user-frontend/

# 构建管理后台
cd ../manager
npm ci
npm run build
cp -r dist/* /var/www/deskpet/manager/
```

#### 或者在本地构建后上传

```bash
# 本地构建
cd deskpetConsole/user-frontend
npm ci && npm run build

cd ../manager
npm ci && npm run build

# 上传到服务器
scp -r deskpetConsole/user-frontend/dist/* user@your-server:/var/www/deskpet/user-frontend/
scp -r deskpetConsole/manager/dist/* user@your-server:/var/www/deskpet/manager/
```

### 4.6 配置 Nginx

```bash
# 创建配置目录
sudo mkdir -p /etc/nginx/conf.d
sudo mkdir -p /etc/nginx/ssl
```

将项目中的 Nginx 配置复制到服务器：

```bash
# 主配置
sudo cp deploy/nginx/nginx.conf /etc/nginx/nginx.conf

# 站点配置
sudo cp deploy/nginx/conf.d/deskpet.conf /etc/nginx/conf.d/deskpet.conf
```

或者从本地 scp：

```bash
scp deploy/nginx/nginx.conf user@your-server:/tmp/nginx.conf
scp deploy/nginx/conf.d/deskpet.conf user@your-server:/tmp/deskpet.conf
# 在服务器上
sudo mv /tmp/nginx.conf /etc/nginx/nginx.conf
sudo mv /tmp/deskpet.conf /etc/nginx/conf.d/deskpet.conf
```

#### 配置域名

将配置中的 `yourdomain.com` 替换为实际域名：

```bash
sudo sed -i 's/yourdomain.com/example.com/g' /etc/nginx/conf.d/deskpet.conf
```

#### 无域名（纯 IP 访问）

如果没有域名，将 `/etc/nginx/conf.d/deskpet.conf` 替换为以下内容：

```nginx
upstream pet_core {
    server 127.0.0.1:8080;
}

upstream pet_ai {
    server 127.0.0.1:8090;
}

# 用户前端 - 端口 80
server {
    listen 80;

    root /var/www/deskpet/user-frontend;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /assets/ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    location /api/ {
        proxy_pass http://pet_core/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /ws {
        proxy_pass http://pet_core/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 3600s;
    }

    location /ai/ {
        proxy_pass http://pet_ai/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}

# 管理后台 - 端口 8088
server {
    listen 8088;

    root /var/www/deskpet/manager;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /assets/ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    location /api/ {
        proxy_pass http://pet_core/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /ws {
        proxy_pass http://pet_core/ws;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_read_timeout 3600s;
    }
}
```

#### 验证并启动 Nginx

```bash
# 测试配置
sudo nginx -t

# 启动/重载
sudo systemctl start nginx
# 或者如果已在运行
sudo systemctl reload nginx
```

### 4.7 配置 HTTPS（推荐）

#### 方式一：Let's Encrypt 免费证书

```bash
# 安装 certbot
sudo apt install -y certbot

# 停止 Nginx 释放 80 端口
sudo systemctl stop nginx

# 申请证书（多个域名）
sudo certbot certonly --standalone \
    -d pet.yourdomain.com \
    -d admin.yourdomain.com \
    -d api.yourdomain.com

# 复制证书
sudo cp /etc/letsencrypt/live/pet.yourdomain.com/fullchain.pem /etc/nginx/ssl/pet.yourdomain.com.pem
sudo cp /etc/letsencrypt/live/pet.yourdomain.com/privkey.pem /etc/nginx/ssl/pet.yourdomain.com.key
# 其他域名同理...
```

编辑 `/etc/nginx/conf.d/deskpet.conf`：
1. 取消各 server 块中 SSL 相关行的注释
2. 注释掉 `listen 80;`
3. 取消底部 HTTP -> HTTPS 重定向的注释

```bash
sudo nginx -t && sudo systemctl start nginx
```

设置自动续期：

```bash
# 测试续期
sudo certbot renew --dry-run

# certbot 安装时已自动配置了 systemd timer 或 cron
# 续期后需要 reload nginx
sudo tee /etc/letsencrypt/renewal-hooks/post/reload-nginx.sh > /dev/null <<'EOF'
#!/bin/bash
cp /etc/letsencrypt/live/pet.yourdomain.com/fullchain.pem /etc/nginx/ssl/pet.yourdomain.com.pem
cp /etc/letsencrypt/live/pet.yourdomain.com/privkey.pem /etc/nginx/ssl/pet.yourdomain.com.key
systemctl reload nginx
EOF
sudo chmod +x /etc/letsencrypt/renewal-hooks/post/reload-nginx.sh
```

#### 方式二：购买的证书

将 `.pem` 和 `.key` 文件放到 `/etc/nginx/ssl/` 目录，文件名与配置中一致。

## 七、防火墙配置

只开放必要端口：

```bash
# Ubuntu (ufw)
sudo ufw allow 22/tcp     # SSH
sudo ufw allow 80/tcp     # HTTP
sudo ufw allow 443/tcp    # HTTPS
sudo ufw allow 1883/tcp   # MQTT（设备接入）
sudo ufw enable

# CentOS (firewalld)
sudo firewall-cmd --permanent --add-port=22/tcp
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --permanent --add-port=443/tcp
sudo firewall-cmd --permanent --add-port=1883/tcp
sudo firewall-cmd --reload
```

如果使用 IP 模式还需要开放管理后台端口：

```bash
sudo ufw allow 8088/tcp
```

云服务器还需要在安全组中放行对应端口。

## 八、常见问题

### Q: pet-core 启动失败，提示数据库连接失败

```bash
docker compose ps db
docker compose logs db
```

db 还在初始化时 pet-core 会连接失败，等 healthcheck 通过后会自动重连。

### Q: Nginx 报 502 Bad Gateway

后端服务还没启动完成，或者端口绑定有问题：

```bash
# 确认后端在监听
curl http://127.0.0.1:8080/actuator/health
# 查看 Docker 日志
docker compose logs pet-core
```

### Q: 前端页面空白

1. 检查 `/var/www/deskpet/user-frontend/` 下是否有 `index.html`
2. 检查 Nginx 配置中 `server_name` 是否与访问域名一致
3. 查看浏览器控制台是否有 JS 报错

### Q: MQTT 设备连不上

1. 确认 1883 端口已在防火墙和安全组中放行
2. `docker compose logs mqtt-gateway`
3. 确认 `gateway-config.yaml` 中 `core.internalBaseUrl` 为 `http://pet-core:8080`

### Q: 前端更新后浏览器还是旧版本

Vite 构建的静态资源文件名带 hash，正常情况下不会有缓存问题。如果 `index.html` 被缓存：

```bash
# 确认 Nginx 没有对 index.html 设置长缓存
# 默认配置只对 /assets/ 设置了 30d 缓存，index.html 不会被缓存
```

## 九、deploy 目录文件清单

```
deploy/
├── docker-compose.yml          # 本地开发用
├── docker-compose.prod.yml     # 生产环境用（不含 Nginx 和前端容器）

```

├── conf.d/
